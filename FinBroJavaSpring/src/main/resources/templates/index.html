    • http://testvps.us.to:8080/api/accounts/

  • GET - /addAccount (@RequestBody Account)
  • GET - /allAccounts
  • GET - /getByAccountId/{accountId} (@PathVariable accountId)
  • GET - /getAllByUserId/{userId} (@PathVariable userId)
  • PUT - /updateAccount (@RequestBody Account)
  • DELETE - /deleteById/{accountId} (@PathVariable accountId)

    • Table name: Account

    Attribute Name Data Type Notes
    id int
    name String
    balance (nullable) float cannot be below 0
    dateCreated (nullable) String format: yyyy-MM-dd HH:mm:ss
    minimumBalance (nullable) float cannot be below 0
    notes (nullable) String cannot exceed 500 characters
    userId int
    categoryId (nullable) int

Possible Error Messages

  • 400 Bad Request - The request could not be understood by the server due to malformed syntax or invalid data.
  • USERNAME_ALREADY_EXISTS - User with the given username already exists.
  • USER_ID_NOT_FOUND - User with the given id does not exist.
  • USERNAME_NOT_FOUND - User with the given username does not exist.
  • EMAIL_ALREADY_EXISTS - Given email already belongs to a User.
  • EMAIL_NOT_FOUND - Given email does belong to an existing User.
  • INVALID_EMAIL_FORMAT - Given email does not fit expected format.
  • MISSING_PARAMETER - Given response body is missing parameter/s.
  • INCORRECT_PASSWORD - Given password is incorrect.
  • BALANCE_BELOW_ZERO - Bank balance is less than zero.
  • BALANCE_BELOW_MINIMUM - Balance is less than the minimum set balance.
  • NOTES_TOO_LONG - Account Notes exceed 500 characters.
  • INVALID_DATE_FORMAT - Date format does not conform to the format yyyy-MM-dd HH:mm:ss.
  • ACCOUNT_ID_NOT_FOUND - Account with given id does not exist.

Create User

Path

/addUser

Description

Creates a new Users and stores in the database.

HTTP Method

POST

Request Body Example

{
    "name": "Bob",
    "surname": "Smith",
    "age": 35,
    "username": "bob.s",
    "email": "bob.s@example.com",
    "password": "hashed_password2"
}

Response Example

{
    "id": 2,
    "name": "Bob",
    "surname": "Smith",
    "age": 35,
    "username": "bob.s",
    "email": "bob.s@example.com",
    "password": "hashed_password2"
}
